# E-com Prophet

## Описание проекта

E-com Prophet — это Streamlit-приложение для операционного анализа продаж на маркетплейсах. Оно помогает продавцу оценивать спрос, планировать поставки, анализировать рекламу и промо-активности, а также получать рекомендации по цене для SKU.

Текущая версия работает с CSV-данными и использует эвристические расчеты на базе `pandas` и `numpy`. Это практический аналитический инструмент, а не полноценная ML-система прогнозирования временных рядов.

## Основные возможности

1. Прогноз спроса
   - расчет прогноза продаж по SKU;
   - горизонт прогноза 14 или 30 дней;
   - учет последних продаж, конверсии, рейтинга, промо-фактора и сезонных признаков, если они есть в CSV.

2. Планирование поставок
   - расчет рекомендуемого количества к поставке;
   - учет текущего и зарезервированного остатка, если эти поля есть в данных;
   - фильтрация по минимальной маржинальности.

3. Анализ рекламы и промо
   - оценка рекламных расходов, ROI, конверсии и маржи после рекламы;
   - рекомендации по усилению, наблюдению или сокращению продвижения.

4. Рекомендации по цене
   - расчет минимальной цены для целевой маржи;
   - оценка текущей цены, комиссии, себестоимости и прибыльности;
   - рекомендации по повышению, удержанию или пересмотру цены.

Каждая функция поддерживает два режима:

- анализ всех товаров, подходящих под выбранные фильтры;
- анализ одного выбранного товара/SKU.

## Структура проекта

```text
app.py                     # основной Streamlit-интерфейс

```

## Локальный запуск

Этот раздел описывает универсальный запуск проекта после клонирования репозитория. Он не привязан к конкретному пути на компьютере разработчика.

Клонируйте репозиторий и перейдите в папку проекта:

```powershell
git clone https://github.com/89675706969zhw-lang/E-com-Prophet.git
cd E-com-Prophet
```

Создайте виртуальное окружение:

```powershell
python -m venv .venv
```

Активируйте окружение в Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

Для macOS или Linux используйте:

```bash
source .venv/bin/activate
```

Установите зависимости:

```powershell
python -m pip install -r requirements.txt
```

Запустите приложение:

```powershell
streamlit run app.py
```

После запуска Streamlit покажет локальный адрес, обычно:

```text
http://localhost:8501
```

## Формат данных

Приложение принимает CSV-файлы с данными о продажах, товарах, ценах, расходах и остатках. Система умеет автоматически сопоставлять разные названия колонок с внутренней схемой.

Наиболее важные поля:

```text
product_sku
product_name
marketplace
category
order_date
quantity_sold
final_price
cost
commission_rate
advertising_cost
profit
margin_percent
profit_per_item
current_stock
reserved_stock
conversion_rate
product_rating
is_promo
marketing_roi
```

Если часть расчетных полей отсутствует, приложение пытается вывести их из доступных данных. Например, `revenue`, `profit`, `margin_percent`, `profit_per_item`, `conversion_rate` и `marketing_roi` могут быть рассчитаны автоматически при наличии исходных колонок.

## Проверка и тестирование

Проверка синтаксиса:

```powershell
python -m py_compile app.py config.py data_utils.py models.py schema_utils.py ui.py
```

Основные бизнес-функции:

```text
estimate_sku_forecast  # прогноз спроса
build_supply_plan      # план поставок
analyze_promo          # анализ рекламы и промо
recommend_prices       # рекомендации по цене
```

Проект проверялся в двух режимах: один товар и несколько товаров. Для проверки использовались основной файл `transactions_optimized.csv` и CSV-файлы из папки `test_csv`.

## Деплой

Деплой — это запуск приложения на сервере или облачной платформе, чтобы им можно было пользоваться не только на локальном компьютере.

### Вариант 1. Запуск на сервере вручную

1. Скопируйте проект на сервер или клонируйте репозиторий.
2. Установите Python 3.10+.
3. Установите зависимости:

```bash
python -m pip install -r requirements.txt
```

4. Запустите Streamlit:

```bash
streamlit run app.py --server.address 0.0.0.0 --server.port 8501
```

5. Откройте порт `8501` в настройках сервера или проксируйте приложение через Nginx.

### Вариант 2. Streamlit Community Cloud

1. Загрузите проект в GitHub-репозиторий.
2. Убедитесь, что в корне есть:

```text
app.py
requirements.txt
```

3. В Streamlit Community Cloud выберите репозиторий.
4. Укажите главный файл:

```text
app.py
```

5. Запустите деплой.

Важно: файл `transactions_optimized.csv` достаточно большой, поэтому перед публикацией стоит проверить лимиты выбранной платформы. Если платформа ограничивает размер репозитория или время запуска, лучше хранить данные отдельно и загружать CSV через интерфейс приложения.

## Ограничения текущей версии

- прогноз спроса является эвристическим, а не полноценной моделью временных рядов;
- качество результатов зависит от полноты и качества CSV-файла;
- часть текстов интерфейса в исходных файлах повреждена из-за кодировки и требует отдельного исправления;
- проект пока не содержит автоматических `pytest`-тестов.
